## Project Overview

Weasis is an open-source DICOM viewer for medical image visualization, designed for both standalone and web-based use. It's a Java/Swing desktop application built on OSGi modular architecture.

**Key Technologies**: Java 24, Maven, OSGi, Swing (FlatLaf), MigLayout, OpenCV, dcm4che, JogAmp (JOGL for 3D rendering)

## Repository Structure

Monorepo with multiple Maven modules:

### Core Modules
- **`weasis-core/`** - Core API, GUI utilities, image processing, media handling
- **`weasis-base/`** - Base viewer components (2D viewer, explorer, UI components)
- **`weasis-dicom/`** - DICOM support modules:
  - `weasis-dicom-codec` - DICOM image codec
  - `weasis-dicom-viewer2d` - 2D DICOM viewer
  - `weasis-dicom-3d` - 3D volume rendering
  - `weasis-dicom-sr` - Structured Reports
  - `weasis-dicom-rt` - Radiotherapy support
  - `weasis-dicom-qr` - Query/Retrieve
  - `weasis-dicom-send` - DICOM Send (storeSCU)
  - `weasis-dicom-explorer` - DICOM file explorer
- **`weasis-imageio/`** - Image I/O support
- **`weasis-opencv/`** - OpenCV integration with native bindings
- **`weasis-launcher/`** - Application launcher
- **`weasis-distributions/`** - Build scripts for installers
- **`tests/`** - Aggregated test module with JaCoCo coverage

### Build System
- **`weasis-parent/pom.xml`** - Parent POM with shared configuration, dependency management
- **`pom.xml`** (root) - Aggregator POM listing all modules

## Build Commands

```bash
# Clean install (requires JDK 24)
mvn clean install

# Build with code coverage
mvn -Pcoverage -B verify

# Run tests only
mvn test

# Run single test class
mvn test -Dtest=ClassName

# Format code (spotless)
mvn spotless:apply

# Build installer (produces .msi, .pkg, .deb, .rpm)
mvn -P compressXZ -f weasis-distributions clean package
```

## Running Tests

Tests use JUnit 6 / Jupiter. The `tests` module aggregates coverage from all modules using JaCoCo.

```bash
# Run all tests
mvn test

# Run specific test
mvn test -pl weasis-core -Dtest=org.weasis.core.api.media.data.TagWTest

# Run with coverage report (generates in tests/target/site/jacoco-aggregate/)
mvn -Pcoverage verify
```

## Key Architecture Concepts

### OSGi Module System
Weasis uses Apache Felix OSGi framework. Each module is an OSGi bundle with:
- Manifest generated by bnd-maven-plugin
- Declarative Services (DS) annotations for component registration
- Service-based architecture for extensibility

### Plugin Architecture
Weasis supports plugins via OSGi bundles. See `archetype/` for Maven archetypes:
- `weasis-plugin-base-viewer-archetype` - Non-DICOM viewer plugins
- `weasis-plugin-dicom-viewer-archetype` - DICOM viewer plugins

### Key Dependencies
- **dcm4che** (via weasis-dicom-tools) - DICOM toolkit
- **OpenCV** - Image processing (custom native builds per platform)
- **JogAmp** - OpenGL/JOGL for 3D rendering
- **FlatLaf** - Look and feel
- **MigLayout** - Layout manager

### Native Libraries
Platform-specific native libraries are in separate modules:
- `weasis-opencv-core-${os}-${arch}` - OpenCV native bindings
- `jogamp-*` - JOGL native bindings
